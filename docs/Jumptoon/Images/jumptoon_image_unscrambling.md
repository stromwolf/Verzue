# Jumptoon Image Unscrambling Technical Overview

This document provides a comprehensive technical breakdown of how Jumptoon manga/webtoon images are scrambled and the process used by the Verzue provider to reconstruct them into their original form.

## 1. Overview
Jumptoon employs a mesh-based "padded atlas" scrambling method. Instead of simply rearranging tiles, the images are split into thin vertical strips, padded with extra pixels, and then shuffled using a deterministic algorithm. To view the image correctly, the strips must be extracted from their padded blocks and reassembled in the correct order.

## 2. Scrambling Pattern (Version V2)
The current scrambling configuration (V2) uses the following parameters extracted from the Jumptoon JavaScript source:

| Parameter | Value | Description |
| :--- | :--- | :--- |
| `splitWidth` (a) | 20px | The actual content width of a strip. |
| `paddingWidth` (o) | 15px | Padding added to both the left and right sides of each strip. |
| `blankWidth` (n) | 1px | A minor gap/gutter between blocks in the raw image. |
| **`l_block`** | **51px** | Total width of one block in the raw image ($20 + 1 + 15 \times 2 = 51$). |

### Raw Image Structure
The scrambled image fetched from the server consists of these 51px blocks. 
- A raw block at index $i$ starts at $X = i \times 51$.
- Within that block, the actual image data starts at $X_{data} = (i \times 51) + 15$ and spans 20px.

---

## 3. Seed Calculation
The shuffle order is deterministic and depends on a seed value unique to each episode.

### Strategy
The seed is generated by summing the character code points of a string composed of the `seriesId` and the `episodeNumber`.

```python
seed_str = f"{series_id}:{episode_number}"
seed = sum(ord(c) for c in seed_str)
```

For example, if the series ID is `JT123` and the episode number is `45`, the seed string is `"JT123:45"`.

---

## 4. Deterministic Shuffling (LCG)
Jumptoon uses a Linear Congruential Generator (LCG) to generate the shuffle sequence. This LCG matches the standard implementation found in their JavaScript client.

### LCG Formula
- **Multiplier**: `1664525`
- **Increment**: `0x3C6EF35F` (1013904223)
- **Modulus**: `0x100000000` ($2^{32}$)

```python
state = (1664525 * state + 0x3C6EF35F) % 0x100000000
```

### Shuffle Logic (Fisher-Yates)
1. Initialize an array `d` with indices `[0, 1, ..., num_blocks - 1]`.
2. Determine the shuffle limit. If the image has a remainder block (see below), the last block is excluded from the shuffle.
3. For each index $i$ from `limit` down to `2`:
    - Generate the next LCG state.
    - Calculate a `target` index: `lcg_state % i`.
    - Swap `d[target]` with `d[i - 1]`.

---

## 5. Reconstruction Workflow
The unscrambling process reverses the shuffle and removes the padding.

### A. Calculate Block Counts
- `num_blocks = raw_width / 51`
- `remainder = requested_width % 20` (Only applicable if a specific width was requested via the API).

### B. Generate Inversion Map
The shuffle array `d` tells us where a source block *goes*. We need an inversion map `p` that tells us which source block *belongs* at destination index $i$.

```python
p = [0] * num_blocks
for i, original_pos in enumerate(d):
    p[original_pos] = i
```

### C. Canvas Reassembly
1. Create a new empty canvas:
   - `Width = num_blocks * 20 + remainder`
   - `Height = raw_height`
2. For each index $i$ from `0` to `num_blocks - 1`:
   - Identify source block index: `src_idx = p[i]`
   - Calculate source X: `src_x = (src_idx * 51) + 15` (skip the 15px padding)
   - Calculate destination X: `dest_x = i * 20`
   - Crop a 20px wide strip from the raw image at `src_x`.
   - Paste the strip onto the canvas at `dest_x`.
3. If a `remainder` exists:
   - Extract the final partial block from the end of the raw image and paste it at the end of the canvas.
